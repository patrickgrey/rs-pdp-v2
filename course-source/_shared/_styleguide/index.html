Style Guide

The purpose of these styles is to give a foundation of reusable styles which can be overwritten in order to adjust them where required. Some inline styles have been added, like colours and borders to help better illustrate the core styles but everything related to layout or vertical flow, for example, is done only with core styles. Normally local styles or a utility like Tailwind would make these adjustments.

If you have a short page (where the footer doesn't reach the bottom of the window, add the .ians-fix-short class to the body tag and remove the ScrollToTop component.

Layouts

Layouts restrict width horizontally and adjust responsively. These are default widths but we are not restricted to them, they can be any width. The default widths speed up development and aid consistency but the ability to have any width means we can adapt to content where needed instead of adapting content to widths.
